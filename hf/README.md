# hf/

Organizer-only tools for managing the leaderboard HuggingFace Space.

**Participants do not need anything in this folder.** To download benchmark data use [`scripts/run_lmudata.py`](../scripts/run_lmudata.py). To submit predictions go to **https://vantage-bench.org/submit**.

---

## Files

### `app.py`

The Gradio leaderboard Space that renders `data/leaderboard.json`. Deployed at  
[https://huggingface.co/spaces/nvidia/PhysicalAI-VANTAGE-Bench](https://huggingface.co/spaces/nvidia/PhysicalAI-VANTAGE-Bench).

It is UI-only — it reads a pre-built artifact (`data/leaderboard.json`) and does not re-score submissions.

### `down.py`

Low-level HuggingFace `snapshot_download` wrapper. Useful for bulk-pulling specific files from a private dataset repo. Participants should use `scripts/run_lmudata.py` instead.

### `up.py`

Uploads a local `leaderboard.json` to the HF Space repo. Run by the organizer after updating scores:

```bash
python hf/up.py
```

Requires `HF_TOKEN` set or `hf auth login` completed. The `REPO_ID` in the script must point to the live Space.

---

## Leaderboard pipeline (organizer flow)

```
outputs/<model>/<eval_id>/          ← generated by run.py
        │
        ▼
parser/get_all_outputs.py           ← compiles JSON metric files into vlmevalkit_outputs.json
        │
        ▼
parser/prepare_outputs_hf.py        ← merges into hf/leaderboard.json (the leaderboard schema)
        │
        ▼
hf/up.py                            ← uploads leaderboard.json to the HF Space
        │
        ▼
hf/app.py (Space)                   ← renders the live leaderboard
```

See [`../parser/README.md`](../parser/README.md) for the compilation step details.
